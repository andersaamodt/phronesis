# Lessons

Promoted lessons are short durable rules extracted from repeated audit or
implementation failures.

Use this area for lessons that have not yet grown into full standards or
decision records, but are important enough to preserve centrally.

Guidelines:

- Keep each lesson concrete and reusable.
- Prefer one rule per bullet or one lesson per file.
- Promote a lesson into `standards/` or `decisions/` once it becomes policy.
- Keep repo-specific debugging history in local repos; only promote lessons that generalize across the wizardry family.

- `boundaries-and-inputs.md`
- `gui-and-hosts.md`
